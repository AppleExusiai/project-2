# 🔧 高校防诈骗宣传平台 - 故障排除指南

## 常见问题解决方案

### 问题 1️⃣: "无法创建用户" 错误

#### 可能原因和解决方案：

**原因 A: 用户名或邮箱已被注册**
- 错误信息：`用户名或邮箱已被注册`
- 解决方案：使用不同的用户名或邮箱进行注册
- 检查方法：在测试工具中填入用户名/邮箱并检查是否已存在

**原因 B: 表单验证失败**
- 验证规则：
  - 用户名：3-16个字符，只能包含字母、数字和下划线
  - 邮箱：必须是有效的邮箱格式 (例如: user@example.com)
  - 密码：至少6位字符
- 错误信息示例：
  - `用户名长度必须在3-16个字符之间`
  - `用户名只能包含字母、数字和下划线`
  - `邮箱格式不正确`
  - `密码长度至少为6位`

**原因 C: 两次密码输入不一致**
- 错误信息：`两次输入的密码不一致`
- 解决方案：确保"密码"和"确认密码"字段完全相同

**原因 D: 填写的必填项不完整**
- 错误信息：`请填写所有必填项`
- 必填项：用户名、邮箱、密码、确认密码
- 可选项：真实姓名、学号、所在学院、专业

**原因 E: 服务器连接问题**
- 症状：长时间无响应，无法看到任何错误信息
- 解决方案：
  1. 检查服务器是否正在运行：`npm start`
  2. 确认服务器地址正确：`http://localhost:3000`
  3. 检查防火墙设置

## 📋 完整的注册流程

### 前端流程
```
用户填写表单
    ↓
前端验证（检查格式、必填项）
    ↓
表单验证通过
    ↓
发送POST请求到 /api/auth/register
    ↓
等待服务器响应
    ↓
显示成功/失败消息
```

### 后端流程
```
收到注册请求
    ↓
验证所有必填字段
    ↓
验证邮箱格式
    ↓
验证密码长度和一致性
    ↓
使用bcryptjs加密密码
    ↓
尝试插入数据库
    ↓
检查UNIQUE约束（用户名/邮箱）
    ↓
返回成功或失败响应
```

## 🔍 调试技巧

### 1. 查看浏览器控制台
打开浏览器开发工具 (F12)，查看 `Console` 标签：
- JavaScript错误
- 网络请求的详细信息
- 调用堆栈跟踪

### 2. 查看网络请求
在浏览器开发工具中查看 `Network` 标签：
- 请求URL是否正确
- 请求方法（应该是 POST）
- 请求体是否包含所有字段
- 响应状态码和内容

### 3. 查看服务器日志
运行服务器时会看到详细的日志：
```
收到注册请求: { username: '...', email: '...' }
密码加密成功
开始插入数据库...
用户注册成功: ...
```

如果看到错误，日志会显示具体的错误信息。

### 4. 直接用curl测试API
```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "email": "test@example.com",
    "password": "password123",
    "confirmPassword": "password123"
  }'
```

### 常见错误和含义

| 错误 | 含义 | 解决方案 |
|------|------|--------|
| Cannot POST /api/auth/register | API路由不存在 | 检查服务器是否启动，路由是否正确 |
| CORS error | 跨域请求被拒绝 | 检查服务器CORS配置 |
| 用户名或邮箱已被注册 | 账户已存在 | 使用不同的用户名/邮箱 |
| 请求超时 | 服务器无响应 | 检查服务器是否正在运行 |
| JSON解析错误 | 请求格式不正确 | 确保使用正确的JSON格式 |
